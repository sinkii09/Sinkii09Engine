How Sinkii09 Engine ensures services initialize in the correct order

1

Registration Phase

Services declare their dependencies using attributes:

[EngineService(RequiredServices = new[] { typeof(IUIService) })]

ServiceContainer captures these dependencies:

registration.RequiredDependencies = engineAttr?.RequiredServices
2

Graph Building

Build a directed graph of dependencies:

  • Each service becomes a node
  • Dependencies become edges
  • Detect circular dependencies
var dependencyGraph = container.BuildDependencyGraph();
3

Topological Sort

Calculate initialization order using Kahn's algorithm:

  • Find services with no dependencies
  • Process them first
  • Remove edges and repeat
var initOrder = await graph.GetInitializationOrderAsync();
4

Sequential Init

Initialize services in calculated order:

  • Dependencies init first
  • Dependents can access them
  • Guaranteed availability
foreach (var service in initOrder) { await service.InitializeAsync(); }

DialogueService declares RequiredServices = [IUIService]. This ensures:

// DialogueService.cs - Correct dependency declaration
[EngineService(ServiceCategory.UserInterface, ServicePriority.High,
    Description = "Manages character dialogue, player choices, and narrative progression",
    RequiredServices = new[] { typeof(IUIService) })]  // Only truly required service
public class DialogueService : IDialogueService
{
    public async UniTask<ServiceInitializationResult> InitializeAsync(IServiceProvider provider, ...)
    {
        // Required - will fail if missing
        _uiService = provider.GetService(typeof(IUIService)) as IUIService;
        if (_uiService == null)
            return ServiceInitializationResult.Failed("UIService dependency is required");
        
        // Optional - null-safe operations
        _audioService = provider.GetService(typeof(IAudioService)) as IAudioService;
        _saveLoadService = provider.GetService(typeof(ISaveLoadService)) as ISaveLoadService;
    }
}
